# E-Commerce Backend API

A robust, N-Tier E-Commerce backend API built with .NET 10, SQL Server, and Entity Framework Core. This project provides a fully functional backend for an e-commerce platform, featuring secure authentication, product catalog management, shopping cart operations, and order processing workflows.

## Architecture

The application follows an N-Tier architecture to ensure separation of concerns, scalability, and maintainability:

*   **Presentation Layer (API):** ASP.NET Core Web API handling HTTP requests, routing, and exposing endpoints.
*   **Business Logic Layer (BLL):** Contains all business rules, services, and DTOs (Data Transfer Objects). Responsible for processing data between the API and DAL.
*   **Data Access Layer (DAL):** Manages all database interactions using Entity Framework Core. Includes Models/Entities, Repositories, and the Unit of Work pattern.

## Core Features

*   **Authentication & Authorization:** Secure user authentication using JWT (JSON Web Tokens) and ASP.NET Core Identity. Includes role-based access control (e.g., Admin vs. Customer).
*   **Product Management:** Full CRUD operations for products and categories. Supports features like filtering, pagination, and local image file storage.
*   **Shopping Cart:** robust cart management system allowing users to add, update, and remove items from their shopping baskets.
*   **Order Processing:** Workflows for converting shopping carts into finished orders.

## Technologies Used

*   **Framework:** .NET 10.0
*   **Language:** C#
*   **Database:** SQL Server
*   **ORM:** Entity Framework Core 10.0.5
*   **Authentication:** ASP.NET Core Identity, JWT Bearer Authentication
*   **API Documentation:** Postman / OpenAPI

## Getting Started

### Prerequisites

*   [.NET 10.0 SDK](https://dotnet.microsoft.com/download/dotnet/10.0)
*   [SQL Server](https://www.microsoft.com/en-us/sql-server/sql-server-downloads) (Express or fully fledged)
*   Your preferred IDE (Visual Studio, VS Code, Rider, etc.)

### Setup Instructions

1.  **Clone the repository:**
    ```bash
    git clone <repository-url>
    cd ApiFinalProject
    ```

2.  **Configure the Database connection:**
    Update the `appsettings.json` file in the `ApiFinalProject.API` project with your SQL Server connection string.
    ```json
    {
      "ConnectionStrings": {
        "DefaultConnection": "Your Connection String"
      }
    }
    ```

3.  **Apply Database Migrations:**
    Navigate to the API project directory and run the Entity Framework Core tools to update the database.
    ```bash
    cd ApiFinalProject.API
    dotnet ef database update --project ../ApiFinalProject.DAL
    ```

4.  **Run the Application:**
    Start the API server.
    ```bash
    dotnet run
    ```

5.  **Test the Endpoints:**
    Import the OpenAPI schema into **Postman** (or your tool of choice) or manually configure the requests to test the REST endpoints interactively. The OpenAPI schema is typically generated by the application for easy importing.

## API Endpoints

Below is a high-level overview of the main REST endpoints exposed by this project:

### Authentication (`/api/Auth`)
*   `POST /api/Auth/register` - Register a new user account.
*   `POST /api/Auth/login` - Authenticate and receive a JWT token.

### Products (`/api/Products`)
*   `GET /api/Products` - Retrieve a paginated/filtered list of products. *(Public)*
*   `GET /api/Products/{id}` - Retrieve a single product by ID. *(Public)*
*   `POST /api/Products` - Create a new product. *(Requires Auth)*
*   `PUT /api/Products/{id}` - Update an existing product. *(Requires Auth)*
*   `DELETE /api/Products/{id}` - Delete a product. *(Requires Auth)*
*   `POST /api/Products/{id}/image` - Upload an image file for a specific product. *(Requires Auth)*

### Cart (`/api/Cart`) - *All require Auth*
*   `GET /api/Cart` - Retrieve the current user's shopping cart.
*   `POST /api/Cart` - Add an item to the shopping cart.
*   `PUT /api/Cart` - Update the quantity of a cart item.
*   `DELETE /api/Cart/{productId}` - Remove an item from the cart.

### Categories & Orders
Similar CRUD operations are structured following standard REST patterns under `/api/Categories` and `/api/Orders`. For the exact request/response schemas, you can import the generated OpenAPI document into Postman.
